# Security Policy

## Supported Versions

We provide security updates for the following versions of `mdwow`:

| Version | Supported          |
| ------- | ------------------ |
| < 1.17.x| :x: No             |
| 1.17.x  | :white_check_mark: Yes |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please **do not open a public issue**. Instead, report it privately using one of the following methods:

1.  **GitHub Private Vulnerability Reporting**: Use the "Report a vulnerability" button under the "Security" tab of the repository.
2.  **Email**: Contact the maintainer directly at [tianhaozhou95@icloud.com](mailto:tianhaozhou95@icloud.com).

Please include the following in your report:
- A detailed description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact if exploited.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Disclosure Policy

When a vulnerability is reported, we follow these steps:
1.  Verify the vulnerability and determine its severity.
2.  Work on a fix in a private branch.
3.  Release a new version with the patch.
4.  Publicly disclose the vulnerability with credit to the reporter (if desired).

Thank you for helping keep `mdwow` secure!
